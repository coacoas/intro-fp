What is Functional Programming?



Programming with Functions

What is a function?



A pure function f(A)⇒B is a mapping of every value of type A to exactly one value of type B

...and nothing else

No side-effects

  • Reading/Writing Files
  • Assigning to variables
  • Setting fields on variables
  • Throwing exceptions
  • etc.

Referential Transparency

An expression e is referentially transparent if all occurrences of e can be replaced by the result of evaluating e without affecting the meaning of the program.


A function f is pure if the expression f(x) is referentially transparent for all referentially transparent x.

Functional Programming is a discipline

  • Only referentially transparent expressions.
  • Only pure functions.
  • No side-effects.
  • No exceptions.

How do we do anything?

  • Looping without variables?
  • Data that changes over time?
  • Error handling without exceptions?
  • I/O without... I/O?

Higher Order Functions

A higher order function is a function that takes a function as an argument, or returns a function as the result.

Looping

Looping is a specialized form of linear recursion.

A linearly recursive function calls itself only once.

A tail-recursive function only calls itself as the last step.

Demo - Factorial

Tail Recursion

def fact(n: Int): Int = if (n <= 1) 1 else n * fact(n - 1)
						

							fact(5) = 
							5 * fact(4) = 
							5 * 4 * fact(3) =
							5 * 4 * 3 * fact(2) =
							5 * 4 * 3 * 2 * fact(1) =
							5 * 4 * 3 * 2 * 1 = 
							5 * 4 * 3 * 2 = 
							5 * 4 * 6 =
							5 * 24 = 
							120
						
def fact(acc: Int, n: Int): Int = if (n <= 1) acc else fact(n * acc, n - 1)

							fact(1, 5) =
							fact(5, 4) = 
							fact(20, 3) = 
							fact(60, 2) = 
							fact(120, 1) = 
							120

Immutability gives us lots of options.

Persistent data structures use data sharing to help

Linked Lists

val x = List(3,2,1)  // 3 :: 2 :: 1 :: Nil

val y = 5 :: 4 :: x

Error handling

Folding with Option[A] or Either[A,B] values

Option[A] indicates that an operation may fail.

Either[A,B] indicates that an operation may fail, and will return the successful value or the error.

I/O

  • Delay processing I/O to "the end of the universe"
  • Create a data structure that defines what needs to be done.
  • The last step of the program is to fold over that data structure.
